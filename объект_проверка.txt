Единичный элемент

- исходное состояние
- воздействие
- логи
- конечное состояние
- событие окончания

состояние 
- настройки, соответствуют названиям и значениям в ПО Guard Light (GC)
- ноги,  соответствуют названиям в программой конфигурации в ПО Guard Light
сохраняем только то, что нужно
не указанное не должно менять состояние от начала до конца

воздействие 
ноги
- название команды on/off (функции)
- параметры = название ноги в терминах ПО Guard Light
карты
- название команды Wiegand/Dallas_start/Dallas_stop (функции)
- параметры вход/выход,  номер карты
ПО (кнопки, режимы)
- название команды кнопка, режим (функции)
- параметры вход/выход,  вкл/выкл

логи
- массив объектов нога{sost, time} = 0/1 и время до 0,1 сек  от старта проверки
- лог событий контроллера

группы 
проверки собираем в группы
номер [массив номеров или номер], флаг = 0-проверка, 1 группа
{arr:[2,4,7], flag:1}  или  {arr:[1], flag:0}
раскрываем объекты пока не получим последователный список прповерок

--------------------------------------------------------------------------------
ноги - массивы
элемент переферии , номер ноги стенда, щуп, группа, номер ноки контроллера
щупы =  (0к1, 1к8, 3к6) , диод шотки, ограничитель
2 выхода с щупами 1к8 и 3к6, вход/выход с щупом 0к1 образуют группу
Варианты, 3 выхода и 1 вход
группа задается монтажем и записиывается в настройках
у группы свой номер, если нога одна то она получает свой номер группы
с номером группы связываем ногу контроллера
с номером ноги стенда связываем элемент переферии 
элемент переферии  - это входы и выходы с которыми взаимодействует тестируемый контроллер,
по списку из Guard Light - программная конфигурация

группы
3 кросса в ряд
io - out - io  соединяем через платы с резисторами и лиодами, перемычки на внешнюю сторону
и 2 из них с доп контактами для считывателей = 2*Z-2 Base то же соединенные в ряд
Каждому контакту назначаем группу по умолчанию для 3 левых колонок
она равна номеру котакта, в правых номер группы равен номеру контакта + 16
